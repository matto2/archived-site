Story 1, a before and after
Open-in flow from attachment

The flow starts with an email attachment. Jeanette presses and holds to bring up the open-in dialog and select the app...no problem so far.

before:

Next she is asked to choose a source which might be confusing since she's expecting to open a file.

Once a source is selected there's no obvious path to take (she ends up having to click the plus icon to continue).

Next she has the option to upload (underneath create new folder) but is required to name the file to complete the task.

Finally the file gets uploaded but she ends up on the activities screen. Where's the file she just wanted to open in the first place?

after:

A more elegant soltion is to consolidate this flow into one screen. She is not required to change the existing file name and the destination is set to her default folder. Upload button is unambiguous. Options still exist to change the name of the file or destination if she wants to but they are not required.

After uploading the file opens as expected. Time to proofread!
